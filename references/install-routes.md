# 可移植安装路由

先确认项目和目标宿主，再选择路线。以下是工具形态，不是对任何 AI 宿主配置命令的猜测。

| 形态 | 优先证据 | 常见安装选择 | 验证 | 注意事项 |
|---|---|---|---|---|
| Skill | 官方 `SKILL.md`、宿主文档 | 项目级或用户级 skill 目录 | 宿主能发现且可读取 `SKILL.md` | 仅使用已验证的 adapter 写入目录 |
| MCP | 官方 MCP 文档、server 包 | 宿主注册或项目配置 | server 被发现并完成最小调用 | 配置通常涉及命令、环境变量或密钥 |
| CLI | 官方包注册表/README | 项目依赖、用户级或全局安装 | 官方 `--version` / `--help` | 检查 PATH、同名命令和版本 |
| Binary | 官方发布页和校验值 | 用户自选目录 | 官方版本命令 | 核对系统、架构、签名/哈希；不要覆盖未知版本 |
| Desktop app | 官方下载页 | 用户级或系统级安装 | 实际启动与版本信息 | 明确安装器写入范围和权限 |

## 宿主接入

只有 `adapters/` 中具有官方来源和验证步骤的 profile 才能提供平台专用命令。没有匹配 profile 时，选择 generic：保留工具的 CLI/MCP/项目依赖安装，并把宿主接入列为待用户按官方文档完成的步骤。

## 网络与供应链

- 代理、镜像和替代下载源不是默认配置；仅在直连失败且用户同意时使用。
- 执行下载的二进制或安装器前，核对官方来源、版本、系统/CPU 架构和可用的哈希/签名。
- 安装前列明将写入的目录、配置和权限；全局安装及写配置必须由用户选择。
